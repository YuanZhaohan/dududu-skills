# Portability

## Runtime contract

- Require Python 3.9 or newer.
- Use only the Python standard library for OCR HTTP calls, caching, chunking, validation, and rendering.
- Local PDF text fallback optionally uses `pypdf`; install it only when fallback is needed.
- Resolve all paths with `pathlib`; do not assume a shell, drive letter, home directory, or path separator.
- Keep semantic extraction in the current host agent. Do not call Codex, OpenClaw, or another LLM from Python.

## Environment

Set:

```text
PADDLEOCR_ACCESS_TOKEN=<secret>
```

Optional overrides:

```text
PADDLEOCR_DOC_PARSING_API_URL=https://example/layout-parsing
PADDLEOCR_REQUEST_FORMAT=auto
```

Never store a real token in `.env`, examples, logs, JSON, Markdown, or shell history generated by the skill.

## Codex

Keep the skill folder in `$CODEX_HOME/skills/` or invoke it from the checked-out repository when supported. The `agents/openai.yaml` file provides Codex UI metadata but is not needed by the extraction scripts.

## OpenClaw

Install the folder through OpenClaw's local-skill installation command or place it under a configured AgentSkills root. OpenClaw accepts the same `SKILL.md` core. Configure the secret through OpenClaw's environment/secret injection rather than editing files.

OpenClaw can substitute `{baseDir}` in skill instructions. Other hosts should resolve the directory that contains the loaded `SKILL.md`.

## Moving a generated Wiki

The Wiki uses relative links for generated pages and assets. Source PDFs are not copied by default; use `--copy-sources` before moving a self-contained archive. Absolute source paths remain provenance metadata and may become stale after moving.
